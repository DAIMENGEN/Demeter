在回答或者实现与 RUST 相关的问题时，请遵循以下规范：

1. **回答问题的方式**
    - 如果问题涉及代码实现或者错误修复，需要通过 use context7 查询最新的文档，以确保答案的准确性和时效性。

2. **其他注意事项**
    - 确保代码符合RUST项目的编码规范。
    - 合理的目录结构和模块划分。
    - 必须考虑项目后续的可维护性和可扩展性。
    
3. **依赖管理工具**
    - 当前使用的依赖管理工具是 cargo。

4. **项目技术栈**
    - **Web 框架**: 使用 `axum` 作为 Web 框架，提供路由、中间件和 HTTP 处理功能。
    - **数据库**: 使用 `PostgreSQL` 数据库。
    - **数据库操作**: 使用 `sqlx` 作为异步数据库操作库，支持编译时 SQL 验证。
      - 使用 `sqlx::query!` 或 `sqlx::query_as!` 宏进行类型安全的 PostgreSQL 查询。
   
5. **回答简洁性**
    - 避免创建过多的说明性 Markdown 文档，确保回答简洁明了，直接解决问题。
